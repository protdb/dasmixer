## Добавлено (Added)

### API и инфраструктура

#### `api/config.py`
- Новый модуль для системных настроек приложения
- Использует `pydantic-settings` для управления конфигурацией
- Хранит данные в системной папке через `typer.get_app_dir()`
- Функции:
  - `add_recent_project()` - добавление в список последних проектов
  - `update_last_import_folder()` - сохранение последней папки импорта
  - `update_last_export_folder()` - сохранение последней папки экспорта
- Автоматическая загрузка при импорте модуля

### Точка входа

#### `main.py`
- Единая точка входа для GUI и CLI режимов
- Использует Typer для CLI
- Формат: `main.py [FILE_PATH] [COMMAND] [ARGS]`
- Логика: если нет команды → запуск GUI, если есть → CLI
- Регистрация команд: `project`, `subset`, `import`

### CLI команды

#### `cli/commands/project.py`
- Команда `create` для создания нового проекта
- Автоматическое создание группы "Control" по умолчанию
- Подтверждение перезаписи существующего файла
- Интеграция с `api/config.py`

#### `cli/commands/subset.py`
- Команда `add` - добавление группы сравнения
  - Параметры: `--name`, `--details`, `--color`
- Команда `delete` - удаление группы
  - С подтверждением
  - Проверка наличия связанных образцов
- Команда `list` - список всех групп
  - С количеством образцов в каждой
- Асинхронные операции с использованием `asyncio.run()`

#### `cli/commands/import_data.py`
- Команда `mgf-pattern` - импорт MGF по шаблону
  - Использует `utils/seek_files.py`
  - Массовый импорт с прогресс-баром
  - Автоматическое создание образцов
  - Назначение группы
- Команда `mgf-file` - импорт одного MGF файла
  - Явное указание sample_id
  - Назначение группы
- Команды `ident-pattern` и `ident-file` - заглушки
  - Будут реализованы после добавления парсеров идентификаций

### GUI компоненты

#### `gui/components/plotly_viewer.py`
- Универсальный компонент для Plotly графиков
- Отображение статичного PNG в основном UI
- Кнопка "Interactive Mode" → запуск pywebview
- Multiprocessing для независимого процесса просмотра
- Обработка ошибок рендеринга
- Настраиваемые размеры (width, height)

#### `gui/components/progress_dialog.py`
- Модальный диалог с прогресс-баром
- Методы:
  - `update_progress(value, status)` - обновление прогресса
  - `is_cancelled()` - проверка отмены (заготовка)
  - `reset()` - сброс прогресса
- Используется для длительных операций

### GUI приложение

#### `gui/app.py`
- Класс `DASMixerApp` - главный контроллер
- Меню приложения (AppBar)
- Роутинг между представлениями
- Методы:
  - `new_project()` - создание проекта через FilePicker
  - `open_project()` - открытие проекта
  - `close_project()` - закрытие проекта
- Интеграция с конфигурацией
- Асинхронные операции

#### `gui/views/start_view.py`
- Стартовый экран приложения
- Компоненты:
  - Заголовок и описание
  - Кнопки "Create" и "Open"
  - Список последних 10 проектов
- Кликабельные элементы для быстрого открытия

#### `gui/views/project_view.py`
- Контейнер рабочего пространства проекта
- Tabs-интерфейс с 4 вкладками
- Передача Project instance в дочерние компоненты

### Архитектурные решения

1. **Единая точка входа** через `main.py`
   - Упрощает использование
   - Совместимость GUI/CLI

2. **Системная конфигурация** через `typer.get_app_dir()`
   - Кроссплатформенность
   - Автоматическое определение папки

3. **Асинхронность везде**
   - Все операции с Project - async
   - CLI использует `asyncio.run()`
   - GUI использует `page.run_task()`

4. **Multiprocessing для pywebview**
   - Независимый процесс для интерактивных графиков
   - Не блокирует основное приложение

5. **Автоматическая регистрация парсеров**
   - Готово к плагинам
   - Простое добавление новых парсеров
