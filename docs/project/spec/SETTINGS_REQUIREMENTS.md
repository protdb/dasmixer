# Стадия "Настройки, струкутра проекта, плагины"

## Системные настройки

Сейчас заложен функционал работы с файлом конфигурации, в который сохраняются недавние файлы, размеры окна и пр.

Нужно доработать приложение, добавив системное окно "Настройки", доступное из меню (отдельное view, из которого можно вернуться к окну проекта)

В этом окне необходимо сделать ряд настроек, а именно:

- Batch operations limits:
  - Текстовая подсказка в разделе: "Увеличение размера для batch-операций повышает производительность, снижая затраты на чтение/запись данных, но увеличивает использование RAM в моменте. Используйте осторожно" 
  - Spectra file batch size (default=5000)
  - Identification file batch size (default=5000)
  - Identiication processing (ppm and ion match) batch size (default=5000)
  - Protein mapping batch size (default=5000)
- Default Color scheme:
  - Список используемых цветов по умолчанию для tool и subset, возможность указать hex-значение цвета в поле и использвоать предпросмотр
  - Строки с цветами можно добавлять и удалять (формат строки: текстовое поле, квадрат предпросмотра, кнопка "сохранить")

## Механизм плагинов

Должна быть реализована возможность динамической установки модулей двух видов:
- Парсеры идентификаций (api.inputs.peptides)
- Модули отчетов (api.reporting.reports)

Есть схожий по логике функционал импорта спектров, с ним пока так не работаем, оставляем только поддержку MGF.

Плагин должен быть в виде python-скрипта или модуля в .zip-архиве, которые можно разместить в локальной папке приложения (определеяем путь через typer), в поддиректории plugins/inputs/identifications или plugins/reports.
- Добавляем ещё один пункт системного меню: плагины. Также как для настроек - отдельное view. В нём:
  - Подразделы: Identification parsers, Reports
  - В каждом: список плагинов (включая системные), чекбоксы (активен/неактивен), кнопка удаления. отображаются также встроенные модули указанных типов, для них не предусматривается возможность удалить.
  - Добавляется возможность загрузить плагин из файла (раз), открыть директорию плагинов (два).
  - Статус плагинов (включен/выключен) хранится в config.json. Удаление плагина - физическое удаление файла/модуля из директории плагинов. Через диалог подтверждения.
  - Требуется оформить базовый guideline для плагина, что плагин должен наследовать такой-то класс и предоставлять такие-то методы. В файле плагина должен вызываться registry.
  - При этом ошибки при вылете плагина в момент инициализации должны отображаться, но не мешать запуску программы в целом (необходимо обернуть импорт плагина в try...except и корректно обработать ошибки)
