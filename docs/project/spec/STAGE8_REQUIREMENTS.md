# Доработки Plotview и TableView

Сейчас реализован сравнительно универсальный функционал работы с компоновкой "таблица + график", используемый в двух точках системы:
- На вкладке Peptides (`gui/views/tabs/peptides/peptides_tab_new.py`)
- На вкладке Proteins (`gui/views/tabs/proteins/proteins_tab.py`)

Под ними лежит общий модуль TabelAndPlotView, состоящий из отдельных компонентов для Plot и Table (`gui/components/base_table_and_plot_view.py`)

## Доработки BaseTableView и имплементаций

Доработки базового класса

- Функционал настройки столбцов в таблице:
  - Добавить кнопку с иконкой шестеренки над таблицей.
  - По нажатию открывается диалог с чекбоксами для каждой из колонок в таблице.
  - В таблице Отображаются только показанные колонки (без перезапроса датафрейма; `get_data()` должна возвращать все возможные столбцы, которые обрабатываются внутри модуля)
  - Иконка доступна только если отображается не пустая таблица, список колонок, который можно настроить, получаем из columns датафрейма.
- Человекочитаемые заголовки.
  - В TableView должен быть параметр `header_name_mapping: dict[str, str]`, в котором можно указать замены названий для отображения в UI
  - Может не быть задан, тогда названия колонок отображаются как есть
  - В нём могут быть описаны не все колонки, неописанные возвращаются как есть
- Tooltips: вместе с dataframe для основных данных функция `get_data` должна отдавать (опционально) ещё один dataframe - tooltips.
  - В нём - колонки, совпадающие по названию с колонками в основном dataframe, соответствующие по индексу строки в этих колонках.
  - Данные из этого dataframe работают как tooltip'ы, т.е. если пользователь наводит курсор на соответствующую ячейку таблицы, с небольшой задержкой появляется окошко с текстом из tooltips
- Кликабельные значения для фильтрации в таблице
  - Добавляем возможность изменить фильтр прямо из таблицы. Для этого создаем в BaseTableView `column_filter_mapping: dict[str, str]`, где задаются названия параметров фильтра (поле в self.filter) и название столбца (без настройки челоекочитаемости)
  - Для поддержки этой логики делаем более сложную логику работы с фильтрами: словарь filter_controls: `dict[str, ft.Control]`. Этот словарь должен заполняться при построении панели фильтров
  - Добавляем два стандартных метода: get_filters_from_ui(), set_filters_in_ui().
    - Первый уже реализован, как часть интерфейса, но должен быть реализован внутри BaseTableView и собирать self.filter на основе filter_controls
    - Второй реализует обратную логику, из self.filter заполняет значения контролов в UI и обновляет список фильтров, а также перестраивает таблицу по новым фильтрам
  - Добавляем возможность выгрузить таблицу в файл
    - Кнопка Export под таблицей (где пагинация) -> запрос данных с имеющимися фильтрами без паджинации -> через pandas сохраняем как csv или xlsx в зависимости от выбора пользователя.
    - Делаем диалог настроек экспорта: формат (CSV, XLSX) как radiobuttons, чекбокс Technical headers (если установлен, то используем родные названия таблиц, иначе делаем `df.rename(columns=self.header_name_mapping))`

# Изменения в логике работы с PPM и разметкой ионов

Вкладка peptides (`gui/views/tabs/peptides/peptides_tab_new.py`).

- В параметры разметки ионов добавляем параметры:
  - Ignore spectre charges (чекбокс, по умолчанию вкл)
  - Min precursor charge (ввод числа, по умолчанию 1)
  - Max precursor charge (ввод числа, по умолчанию 4)
- В `api/spectra/coverage_worker.py` изменяем логику расчета PPM:
  - используем функцию `calculate_ppm_and_charge` из `utils/ppm.py` для расчета PPM
  - Если Ignore spectre charges установлен в False и `item['charge'] is not None` - передаем в нём min_charge=item['charge'], max_charge=item['charge'], иначе передаем заданное на UI
  - Функция рассчитывает PPM, заряд и теоретическую массу. Последнюю при этом нет нужны считать отдельно
  - Заряд нужно сохранять в поле override_charge
- Улучшить работу с разметкой идентификаций белков (peptide_match)
  - Для разметки белковых идентификаций (peptide_match) нужно реализовать логику, аналогичную текущей для идентификаций: делать поправку на заряд (override_charge), работать с batch'ами. И вынести её из gui.
  - В функции map_proteins (`api/peptides/matching.py`) сделать расчет ppm и теоретической массы с учетом значения override_charge. Это значение необходимо в эту функцию также пробросить, расширив наборы полей
