# Задачи по фазе 3_3. Разработка интерфейса для работы с пептидами

## UI

В рамках этой фазы работаем только с вкладкой Peptides пользовательского интерфейса: `gui/views/tabs/peptides_tab.py`.

Эту вкладку частично переделываем, на ней делаем два блока:
- Загрузка файла последовательностей: выбор файла .fasta, преобразование к списку белков
- Настройки инструментов: список блоков параметров для каждого инструмента
- настройки идентификации ионов: параметры определения разметки ионов
- Выбор оптимальной идентификации: параметры выбора оптимальной идентификации для спектра
- Просмотр идентификаций: та логика, которая представлена сейчас на вкладке (поиск и просмотр графиков идентификаций)

По порядку по блокам:

Загрузка файлов последовательностей: выбор файла (.fasta) из файловой системы, кнопка "Загрузить".

Настройки инструментов: блок со список настроенных в проекте инструментов, для каждого задаются:
- max ppm (по умолчанию 50)
- min score (по умолчанию 0.8)
- min ion intensity coverage (по умолчанию 25%)
- чекбокс (use protein identification from file)
- min protein identity (по умолчанию 0.75)
- чекбокс DeNovo seq correction with search 

Настройки разметки ионов:
- Типы ионов, чекбоксы: a, b, c, x, y, z (по умолчанию: b, y)
- Потери воды (чекбокс)
- Потери NH2
- Treshold (ppm)


Выбор оптимальной идентификации: блок с критерием выбора наилучшей идентификации при совпадении:
- radio button: ppm error, intensity coverage
- Для intensity coverage и отображения
- Кнопка "выполнить сопоставление идентификаций" (запускает расчет белковых идентификаций, определение оптимальной идентификации для каждого спектра)

Блок просмотра идентификаций: содержит фильтры (как сейчас сделано на вкладке) с возможностью построить и просмотреть графики идентификаций для пептида.


Теперь важно: сейчас сразу НЕ делаем логику расчета идентификаций в UI, делаем весь описанный пользоватлеьский интерфейс, функцию-заглушку на загруку данных. При изменени открытой вкладки, обновляем список групп.
