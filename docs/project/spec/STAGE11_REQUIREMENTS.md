# Этап 11, доработка представления образцов, операции над образцами

На вкладке "Samples" в секции образцов (`dasmixer/gui/views/tabs/samples/samples_section.py`) требуется внести ряд изменений.
1) Добавить кнопку Update на самый верх, рядом числовое поле min proteins (по умолчанию 30), min identifications (по умолчанию 1000)
2) Список образцов представить как ft.ExpansionPanelList, где каждому образцу соответствует Ft.ExpansionPanel, в представлении по умолчанию все свёрнуты
3) Для образца заголовок в панели:
   1) Название образца; группа; число файлов спектров; число файлов идентификаций (в сумме, всеми инструментами); число идентификаций; число идентификаций с известным (not null) coverage_percent. число preferred идентификаций; число выявленных белков
   2) В начале строки - маркер: зеленая галочка (`ft.Icons.CHECK_CIRCLE_OUTLINE_OUTLINED`) если есть хотя бы один спектр, число файлов идентификаций = кол-во инструментов * на число спектров, кол-во identifications и preferred identifications >= min identifications, количество protein_identification result >= min proteins
   3) Если нет ни одного файла спектра или файла идентификации - красный знак ошибки (`ft.Icons.ERROR_OUTLINE_OUTLINED`)
   4) В остальных случаях желтый знак опасности (`ft.Icons.WARNING_AMBER_OUTLINED`)
   5) Также warning отображаем если есть файлы идентификации с пустым списком идентификаций
   5) Правила применяются с оговоркой: если число preferred идентификаций = 0, то при определении эмблемы не учитывается (до расчета preferred идентификаций у всех ноль, все должны быть зелеными) аналогично с белками
4) Внутри раскрывающегося блока:
   1) Спиок файлов спектров
   2) Список файлов идентификаций для каждого файла спектра с указанием числа идентификаций. Выделен красной рамочкой если число в файле меньше min identifications. Указано также количество preferred identifications
   3) Кнопка удаления файла идентификаций, файла спектра
   4) Кнопки добавления нового файла спектра, новых файлов идентификаций для каждого спектра
   5) Значение параметра additions 
   6) Кнопки действий, в один ряд.
      1) слева: Calculate ions; Select preferred; Match proteins; Protein Identifications; LFQ (надписями)
      2) справа: edit (открывает существующий диалог), mark as outlier; delete (соответствущие иконки с текстом во всплывающей подсказке)
5) Возможные действия:
   6) Удаление спектров и файлов идентификаций: каскадное удаление с подтверждением файла и производных, связанных в проекте через FK
   7) Добавление новых: импорт спектра или идентификации. При импорте идентификации поле с именем образца зафиксировано, поле с выбором инструмента доступно. Для этого нужно доработать существующий диалог выбора единичного файла идентификации, сделав его универсальным с блокирующимися в зависимости от места вызва полями.
   8) Действия с образцом: Calculate ions; Select preferred; Match proteins; Protein Identifications; LFQ. На основе действий, выполняемых на вкладках peptides (см. `dasmixer/gui/views/tabs/peptides/actions_section.py`, `dasmixer/gui/views/tabs/proteins/proteins_tab.py`). Для действий, которые не квантифицируются по образцам сейчас (ion match в частности): требуется модификация процесса, сквозная, для получения из БД и обработки за один заход одного образца. При этом сохраняется логика работы с Batch size. Также здесь требуется рефакторинг, вынести диалоги и процессы в отдельный модуль dasmixer/gui/actions
   9) Отметить как outlier: добавляется столбец outlier для образца, булева величина (1, 0). Миграция данных не требуется. Логику работы с аутлаерами проработаем отделньо в дальнейшем.
6) обновление данных в представлении
   1) выполняется по кнопке update вручную
   2) выполняется после открытия проекта, асинхронно, когда весь остальной UI отрисован.
   3) Также вызывается асинхронно в ряде отделньых случаев, после загрузки спектров, идентификаций, глобальных расчетов (Calculate ions; Select preferred; Match proteins; Protein Identifications)
   4) НЕ выполняется автоматически в любых других условиях
