# Требования к модулям отображения таблиц и графиков

Требуется реализовать общие модули в поьлзовательском интерфейсе для отображения таблиц и построения графиков.

Применяться эти модули должны в двух местах:
- Вкладка "Пептиды": результаты идентификации, поиск и просмотр графиков b/y ионов
- Вкладка "Белки", список идентификаций, просмотр сравнения групп

## Общие доработки

- Таблицы отображаются постранично с паджинацией
- Над таблицами добавляется набор фильтров
- Из таблицы может быть доступно построение графика (кнопка с иконкой графика в строке)
- График имеет свой набор настроек
- График строится с помощью Plotly, его можно посмотреть в интерфейсе (как изображение) и открыть в Webview в интерактивном режиме
- График получает на вход один параметр строки из таблицы, по которому отрисовывается
- Построенные графики можно сохранить в проекте (вручную, нажатием кнопки) и экспортировать.
- Добавляется новая вкладка "Графики", из которой можно экспортировать все сохраненные графики и удалить ненужные

## Детали реализации

СОздаются три базовых класса:

- BasePlotView
- BaseTableView
- BaseTableAndPlotView (объединяющий)

Для BasePlotView переопределяются:
- plot_settings_view: ft.Container, содержащий элементы настройки графика
- plot_settings: dict, содержащий актуальные настройки отображения
- generate_plot: функция, принимающая на вход entityId (ID сущности, для которой график должен быть отрисован) и возвращающая go.Figure

Для BaseTableView переопределяются:
- filter_view: ft.Container, фильтры для отображения таблицы
- filter: dict, содержит актаульные настройки фильтров таблицы
- get_data(limit: int = 100, offset: int = 0) -> pd.DataFrame, получает из БД фильтры
- plot_id_field: str | None, атрибут, название столбца, в котором находится идентификатор, по которому можно построить график. Если None, то не отображется столбец с иконкой графика, график построить нельзя

Для BaseTableAndPlotView задаются:
- table_view: BaseTableView, табилчное представление
- plot_view: BasePlotView, представление графика

BasePlotView и BaseTableView создаются как ExpansionPanelList с двумя ExpansionPanel: настройки и график, фильтры и таблица соответственно.

При инициализации table_view класса передается функция-колбэк, реализуемая в plot_view, получающая на вход ID для построения графика. В свою очередь она вызывает определяемую пользователем функцию generate, а также сохраняет введенные пользователем настройки во внутренний объект plot_settings

Настройки графиков также сохраняются в настройках проекта для замещения значений по умолчанию, как и фильтры таблиц, префиксы: plot_view_{plot_type_name}_, table_view_{table_view_name}

Не делаем реактивного перестроения таблиц и графиков, только по кнопке.

## Предпросмотр

Реализуется по аналогии с таковым для отчетов, генерируется HTML, отображается через pywebview, по возможности переиспользуем написанный код (см. `api/reporting/viewer.py`)

## Экспорт

Должна быть возможность сразу сохранить изображение в .png, сразу после построения. Должна быть возможность перед сохранением задать размер шрифта (для всех компонентов, включая аннотации, задается один шрифт) и размеры изображения в модальном окне. После этого выбирается путь для сохранения. Имя по умолчанию: `{plot_name}_{entity_ID}_{datetime}.png`

## Сохранение в проекте

Создаем отдельную таблицу saved_plots со столбцами:
- created_at
- plot_type
- id
- settings (JSON с параметрами)
- plot (BLOB с go.Figure как gzipped pickle, см. как реализовано для отчетов)

Создаем отделньую вкладку Plots, где можно:
- просмотреть сохраненные графики
- удалить лишние
- Проставить для каждого галочку
- Экспортировать все выделенные (в Word, см. экспорт отчетов: `api/reporting/base.py`)

## Практическая реализация

### График покрытия b/y ионами, вкладка peptides

В целом сохраняем текущий функционал, заменяя на универсальную реализацию.

Настройки графика дублируем из блока "Ion Matching Settings". Фильтры таблицы берем из Search and view Identifications (там же).
См тут: `gui/views/tabs/peptides/peptides_tab_new.py` и по импортам здесь же

### Графики концентрации белков

Табилца есть на вкладке Proteins (`gui/views/tabs/proteins/proteins_tab.py` + импорты). Таблицу делаем в двух вариантах, настойки фильтрами:
- Identifications (примерно как сейчас, показываем каждую строку таблицы peptide_identifications)
- Statistics. Здесь делаем новый метод в проекте, который показывает ряд аггрегатов по каждому белку, появившемуся в идентификациях, а именно:
  - protein_id
  - gene
  - fasta_name (урезано до 30-ти символов)
  - samples (кол-во образцов, в которых встречен)
  - subsets (кол-во групп, в которые попадают образцы с этим белком)
  - PSMs (кол-во идентификаций пептидов из peptide_match)
  - unique (кол-во идентификаций с unique_evidence=True)

Переключение режима по кнопке, кнопка меняет набор фильтров

Фильтры:
- Режим "Identifications"
  - Sample
  - Subset
  - Length
  - protein_id
  - gene
  - По части fasta_name
- Режим "Statistics"
  - Protein_id
  - gene
  - Часть fasta_name
  - Samples (Число, не менее чем...)
  - Subsets (Число, не менее чем...)

График: соотношение концентраций по группам, реализуется как boxplot/violin plot (переключение на усмотрение пользователя)

Настройки:
- Algorithm (метод определения концентрации: NSAF/emPAI/iBAQ/Top3, см. параметры расчета концентрации), по умолчанию emPAI
- Subsets: чекбоксы выбора групп (отображать только выделенные, по умолчанию все) + для каждой поле с порядковым номером, позволяющим настроить порядок следования на графике
- Plot type: Boxplot / Violin plot
- Include title: чекбокс, если установлен - добавляем на график название
- Remove outliers: чекбокс, удалять ли аутлаеры
- Outlier range: поле ввода значения разницы между аутлаером и медианой (если значение отличается от медианы более чем на указанную величину, оно не отображается на графике)
- Show all dots: чекбокс, если выбран - рядом с box/violin plot'ом показываются облака точек всех значений

Данные можно получить из protein_quantification_result, используем их, присоединяя через Join необходимые данные
