# Этап 9, интеграция обновленного Workflow для идентификаций

Сейчас в системе реализован новый механизм работы с ppm, включающий подбор посттрансляционных модификаций, определение заряда и precursor offset'а.

Реализован новый пайплайн для batch-обработки в файле `api/calculations/spectra/identification_processor.py` на замену тому, который реализован в coverage_worker.

В процессе добавляются два параметра:
- source_sequence: исходная немодифицированная последовательность
- isopote_offset: величина precursor isotope offset

Эти два значения должны сохраняться в identification (пустые по умолчанию, могут быть null; в случае с source_sequence - сохраняется только если отличается от sequence). Нужно добавить эти параметры в табилцу identification в Project (миграция данных не требуется, обратную совместимость сейчас не обеспечиваем!)
