Задача текущего этапа - собрать полную и подробную документацию по системе. Тщательно и полностью изучить имеющуюся документацию и код.

Нужно учитывать, что документация сильно устарела. Код считается прироитетным, смотрим на код.

Также на этом этапе нужно составить файлы:
- README.md с описанием проекта и основного функционала для Github
- AGENTS.md с основной информацией для ИИ-агентов разработки. За основу берем docs/MASTER_SPEC.md, убираем разбивку на этапы (они уже так или иначе все завершены), актуализируем, переводим на английский и несколько упрощаем.

Документация в целевом виде должна включать:

`docs/api`: описание API (код в `dasmixer/api`). Разделы:
- project
  - Архитектура: core, mixin'ы, dataclasses, вспомогательные функции. Описательный момент
  - Ключевой API:
    - low-level вызовы (_executemany, _fetchall, ...)
    - Работа с основными сущностми (образцы, инструменты)
  - Mixin API:
    - По каждому Mixin'у, описание
  - Dataclass'ы: описание каждого
- Модули:
  - inputs, reports - внутренняя логика работы, логика работы registry. Документировать встроенные отчеты здесь не требуется
  - calculations: описываем существующие модули и вызовы, даём детали по внутренней логике вызовов.
- Плагины: разработка плагинов, общий подход к их созданию
`docs/user`
  - Документация для пользователя:
    - Логика заполнения форм, создание проекта, работа с группами (subset), образцами, добавление файлов идентификации, правила работы с шаблонами
    - Логика настройки системы: конфигурация, Batch-операции, параметры;
    - Работа с идентификациями, в первую очередь:
      - Значение параметров
      - Логика работы системы в каждом из процессов
      - Как работает построение графиков ионов
    - Работа с белками:
      - Алгоритмы LFQ, рекомендации по использованию
    - Отчеты:
      - Общая логика работы с отчетами: сохранение, интерактивный просмотр, выгрузка. Выбор из файла
    - CLI
`docs/project`
  - Необходимо создать новую версию MASTER_SPEC.md (MASTER_SPEC_NEW.md) с описанием проекта в текущем виде, структуры, размещения файлов с учетом актуального состояния проекта.
    - Описать в этом файле реальную структуру проекта, размещение кода, логику работы, ссылки на конкретные компоненты и документацию
`docs/gui`
  - Описать архитектуру GUI, логику разбивки на компоненты, используемые технологии (акцент на Flet 0.80.5, Интеграцию PyWebView для графиков, логику оптимизации)
`docs/review`
  - При изучении проекта, обращай внимание на неоднозначные и спорные моменты в архитектуре и коде, потенциальные проблемные места.
  - Обрати внимание на неиспользуемые устаревшие файлы - такие есть
  - Подумай над улучшениями.
  - Выпиши всё в этом разделе. Пока не исправляй код, но вынеси сюда все важные наблюдения

Устаревшую документацию и release_notes перенеси в отдельную папку docs/obsolete, в docs не должно остаться ничего лишнего.
