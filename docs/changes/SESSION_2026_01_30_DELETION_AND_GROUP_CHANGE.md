**Дата:** 2026-01-30

2. Добавить возможность изменения группы для образцов

## Реализованный функционал

### 1. Удаление групп (Comparison Groups / Subsets)

**Изменения в `gui/views/tabs/samples_tab.py`:**

- Добавлена кнопка удаления в каждый элемент списка групп (`IconButton` с иконкой `DELETE_OUTLINE`)
- Реализован метод `delete_group(e, group)` с диалогом подтверждения
- Удаление выполняется через `Project.delete_subset(group_id)`
- Обработка ошибок:
  - Если группа имеет связанные образцы → показывается предупреждение
  - Другие ошибки → показывается сообщение об ошибке

**Пользовательский интерфейс:**
- Кнопка удаления отображается справа от каждой группы (красная иконка корзины)
- При клике открывается диалог подтверждения с предупреждением
- После успешного удаления обновляется список групп и образцов

**Валидация:**
- Группу нельзя удалить, если к ней привязаны образцы
- Проверка выполняется на уровне API (`Project.delete_subset`)

### 2. Удаление инструментов (Tools)

**Изменения в `gui/views/tabs/samples_tab.py`:**

- Добавлена кнопка удаления в trailing секцию каждого инструмента
- Реализован метод `delete_tool(e, tool)` с диалогом подтверждения
- Удаление выполняется через `Project.delete_tool(tool_id)`
- Обработка ошибок:
  - Если инструмент имеет связанные идентификации → показывается предупреждение
  - Другие ошибки → показывается сообщение об ошибке

**Пользовательский интерфейс:**
- Кнопка удаления отображается рядом с кнопкой "Import Identifications"
- При клике открывается диалог подтверждения
- После успешного удаления обновляется список инструментов

**Валидация:**
- Инструмент нельзя удалить, если с ним связаны идентификации
- Проверка выполняется на уровне API (`Project.delete_tool`)

### 3. Изменение группы образца

**Изменения в `gui/views/tabs/samples_tab.py`:**

- В метод `refresh_samples()` добавлено создание выпадающего списка (`Dropdown`) для каждого образца
- Dropdown содержит все доступные группы + опцию "None"
- Реализован метод `change_sample_group(e, sample)` для обработки изменения группы
- Изменение группы выполняется через `Project.update_sample(sample)`

**Пользовательский интерфейс:**
- Dropdown отображается в trailing секции каждого образца
- Текущая группа образца выбрана по умолчанию
- При изменении значения группа образца обновляется в базе данных
- После успешного изменения обновляется счетчик образцов в группах

**Логика:**
```python
group_dropdown = ft.Dropdown(
    options=group_options,
    value=current_group_value,  # str(subset_id) или "None"
    width=150,
    height=40,
    on_change=lambda e, s=sample: self.page.run_task(self.change_sample_group, e, s)
)
```

### 4. Исправление импорта идентификаций

**Проблема:**
При импорте идентификаций использовалось поле `parser.spectra_id_field`, которое не было определено в базовом классе парсера.

**Изменения в `api/inputs/peptides/base.py`:**

- Добавлен атрибут класса `spectra_id_field: str = 'seq_no'` (значение по умолчанию)
- Этот атрибут указывает, какое поле использовать для сопоставления со спектрами ('scans' или 'seq_no')

**Изменения в `api/inputs/peptides/MQ_Evidences.py`:**

- Добавлен `spectra_id_field = 'scans'` (MaxQuant использует номера сканов)

**Изменения в `gui/views/tabs/samples_tab.py`:**

В методе `import_identification_files()`:
```python
# Правильная распаковка tuple из parse_batch
async for batch_tuple in parser.parse_batch(batch_size=1000):
    batch = batch_tuple[0]  # Получаем peptide DataFrame
    # ... далее обработка
```

**Логика работы:**
1. Парсер определяет, какое поле использовать для маппинга (`seq_no` или `scans`)
2. При импорте вызывается `Project.get_spectra_idlist(file_id, by=parser.spectra_id_field)`
3. Создается словарь маппинга: `{seq_no/scans -> spectrum_id}`
4. Идентификации маппятся на спектры через `batch['spectre_id'] = batch[parser.spectra_id_field].map(spectra_mapping)`

## Технические детали

### Обновленные методы в SamplesTab

#### `delete_group(e, group)`
- Асинхронный метод удаления группы
- Показывает диалог подтверждения
- Обрабатывает исключения `ValueError` (группа имеет образцы)
- Обновляет UI после успешного удаления

#### `delete_tool(e, tool)`
- Асинхронный метод удаления инструмента
- Показывает диалог подтверждения
- Обрабатывает исключения `ValueError` (инструмент имеет идентификации)
- Обновляет UI после успешного удаления

#### `change_sample_group(e, sample)`
- Асинхронный метод изменения группы образца
- Конвертирует значение dropdown в `int` или `None`
- Обновляет образец через `Project.update_sample()`
- Обновляет счетчики в списке групп
- Показывает уведомление об успешном изменении

#### `refresh_samples()`
- Расширен для создания Dropdown с выбором группы для каждого образца
- Получает список всех групп для заполнения опций
- Устанавливает текущее значение группы образца

### Обработка ошибок

**Удаление с зависимостями:**
```python
try:
    await self.project.delete_subset(group.id)
except ValueError as ex:
    # Показать предупреждение о невозможности удаления
    self.page.snack_bar = ft.SnackBar(
        content=ft.Text(f"Cannot delete: {str(ex)}"),
        bgcolor=ft.Colors.ORANGE_400
    )
```

**Успешное выполнение:**
```python
# Показать уведомление об успехе
self.page.snack_bar = ft.SnackBar(
    content=ft.Text(f"Deleted group: {group.name}"),
    bgcolor=ft.Colors.GREEN_400
)
```

## Использование

### Удаление группы

1. Перейти на вкладку "Samples"
2. В секции "Comparison Groups" найти нужную группу
3. Нажать кнопку с иконкой корзины справа от группы
4. Подтвердить удаление в диалоге
5. Если группа имеет образцы → появится сообщение о невозможности удаления

### Удаление инструмента

1. Перейти на вкладку "Samples"
2. В секции "Identification Tools" найти нужный инструмент
3. Нажать кнопку с иконкой корзины справа от кнопки "Import Identifications"
4. Подтвердить удаление в диалоге
5. Если инструмент имеет идентификации → появится сообщение о невозможности удаления

### Изменение группы образца

1. Перейти на вкладку "Samples"
2. В секции "Samples" найти нужный образец
3. В правой части строки образца выбрать новую группу из выпадающего списка
4. Группа образца обновится автоматически
5. Счетчики образцов в группах обновятся

## Известные ограничения

1. **Групповое удаление**: Пока нет возможности удалить несколько групп/инструментов одновременно
2. **Каскадное удаление**: При удалении группы нельзя автоматически переместить образцы в другую группу - нужно сначала вручную изменить группу у всех образцов
3. **Отмена операций**: Операции удаления необратимы, нет функции отмены
